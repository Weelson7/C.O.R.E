## Infos
- What it is: A static Doom WebAssembly deployment built from `cloudflare/doom-wasm` and served through Nginx.
- What it does: Compiles Doom assets on the target node with `emsdk 3.1.50`, publishes the generated static files at `https://doom.zenith.su`, and validates mesh DNS expectations.
- Why this service exists in C.O.R.E: Provides a lightweight, non-critical validation workload for ingress, TLS, NetBird DNS, and Supervisor-managed service lifecycle conventions.

## Commands
- Start: `cd src/11_Doom && NETBIRD_DEVICE_IP=<mesh-ip> ./deploy.sh`
- Stop: `sudo systemctl stop nginx`
- Restart: `sudo systemctl restart nginx`
- Status: `systemctl status nginx` and `curl -kI https://doom.zenith.su`
- Logs: `sudo tail -n 100 /var/log/nginx/doom.error.log` and `sudo tail -n 100 /var/log/nginx/doom.access.log`
- Edit/Reload: Edit `/etc/nginx/sites-available/doom.zenith.su`, then run `sudo nginx -t && sudo systemctl reload nginx`

## Runtime parameters
- Node(s): Transitional mode, node-local deployment. Any node can host this service per C.O.R.E portability rules.
- Domain: `doom.zenith.su`
- Port(s): `80` (redirect) and `443` (TLS)
- Volumes/Data path: `/var/www/doom` (`/var/www/doom/src` contains built wasm/js/html and `doom1.wad`)
- Environment/config tweaks:
  - Required: `NETBIRD_DEVICE_IP`
  - Optional: `WAD_URL` (defaults to the Doom1.WAD download URL)
  - Build toolchain pinned to `emsdk 3.1.50` at `/opt/emsdk`
- Security constraints:
  - Mesh-scoped DNS expectation through NetBird control plane
  - TLS certificate generated by `mkcert` and installed at `/etc/nginx/ssl/doom.crt` and `/etc/nginx/ssl/doom.key`
  - No local `/etc/hosts` rewrite performed

## Deployment sequence
1. Install runtime/build dependencies (`nginx`, `mkcert`, `git`, `wget`, build-essential toolchain, and Python runtime for emsdk).
2. Clone or update `cloudflare/doom-wasm` into `/var/www/doom` and ensure writable ownership for the invoking operator.
3. Clone or update `/opt/emsdk`, install `3.1.50`, activate it with embedded config, source env, then run Doom wasm build directly on the server.
4. Download `doom1.wad` automatically when missing, then execute `./scripts/clean.sh` and `./scripts/build.sh`.
5. Generate TLS assets using `mkcert`, install certificates into `/etc/nginx/ssl`, write Nginx vhost, validate config, and restart Nginx.
6. Apply final web-root ownership/permissions to `www-data`, verify NetBird client health, and compare domain DNS resolution to `NETBIRD_DEVICE_IP`.
7. Validation checks:
   - `sudo nginx -t` passes
   - `sudo netbird status` succeeds
   - `getent ahostsv4 doom.zenith.su` returns expected mesh IP
   - `https://doom.zenith.su` responds
8. Failure handling notes:
   - If emsdk or build fails, rerun after checking `/opt/emsdk` and build output under `/var/www/doom/src`
   - If DNS validation fails, fix NetBird DNS records centrally and re-check resolution
   - If ingress fails, inspect `/var/log/nginx/doom.error.log` and rerun deploy script

## Files
- `src/11_Doom/deploy.sh`: End-to-end deployment (dependencies, emsdk install/activate, local build, TLS, Nginx, DNS validation).
- `src/11_Doom/wipe.sh`: Removes Doom ingress/content artifacts and optionally purges packages.
- `/etc/nginx/sites-available/doom.zenith.su`: Nginx virtual host for Doom TLS ingress.
- `/etc/nginx/ssl/doom.crt`: Certificate used by Doom HTTPS endpoint.
- `/etc/nginx/ssl/doom.key`: Private key for Doom HTTPS endpoint.
- `/var/www/doom`: Local checkout of `cloudflare/doom-wasm` and generated web assets.
- `/opt/emsdk`: Server-local Emscripten SDK installation pinned to `3.1.50`.
